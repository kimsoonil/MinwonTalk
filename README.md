## 1. 프로젝트 배경

### 현황

한국의 공공 웹사이트는 세계 최고 수준의 전산망을 갖췄으나, 복잡한 인증 절차와 수많은 보안 모듈 설치로 인해 정작 사용자는 접근에 큰 어려움을 겪고 있습니다.

### 목표

"어떤 서류를 어디서 떼야 하나요?"라는 질문에 AI가 답변하고, 인증부터 발급까지 한 곳에서 끝내는 **대화형 민원 창구** 구축.

---

## 2. 핵심 서비스 아키텍처 (The Solution)

### ① LLM 기반 민원 매칭 챗봇

- **기능:** 사용자가 일상어로 질문하면 해당 민원의 정식 명칭과 발급 기관을 즉시 매칭.
- **예시:** "이사가는데 전입신고랑 등본 필요해요" → AI가 정부24의 '전입신고' 서비스와 '주민등록등본 발급' 메뉴를 연결.

### ② 클라우드 기반 '가상 브라우저' 인증 (Zero-Install)

- **핵심 기술:** 사용자 PC에 보안 프로그램을 설치하는 대신, **서버(샌드박스 환경)**에서 보안 프로그램을 미리 구동.
- **장점:** 사용자는 브라우저만으로 모든 인증 과정을 마칠 수 있으며, 개인 PC에 액티브X나 .exe 파일을 설치할 필요가 없음.

### ③ 간편 인증 통합 레이어

- **기능:** 카카오, 네이버, 패스(PASS) 등 민간 간편인증 API를 전면에 배치하여 공동인증서 없이도 모든 민원 처리가 가능하도록 설계.

## 3. 주요 기능 (Key Features)

| 기능명             | 상세 내용                                                                        |
| ------------------ | -------------------------------------------------------------------------------- |
| 맞춤형 서류 패키지 | '연말정산용', '대출 신청용', '취업용' 등 상황별 필요 서류를 한 번에 묶어서 발급. |

## 4. 비즈니스 모델 (Revenue Model)

1. **B2C 서비스료 (Convenience Fee):**
   - 복잡한 서류 뭉치를 한 번에 발급하거나, 인쇄 대행·우편 배송 서비스 이용 시 건당 수수료 발생.
2. **B2B 기업용 솔루션 (SaaS):**
   - 신규 채용이나 금융권 대출 심사 시, 고객에게 '민원똑똑' 링크를 보내 서류를 한 번에 제출받는 기업용 대시보드 판매.
3. **구독형 행정 관리:**
   - 개인 사업자나 소상공인을 대상으로 분기별 세무 신고 서류 및 필수 행정 서류 자동 알림 및 발급 지원.

---

## 5. 단계별 구축 로드맵 (Roadmap)

- **1단계 (MVP):** 가장 수요가 많은 TOP 10 민원(등본, 초본, 인감증명, 가족관계증명서 등)에 대한 챗봇 안내 및 링크 연결 서비스.
- **2단계 (Automation):** 가상 브라우징 기술을 도입하여 보안 프로그램 설치 없이 앱 내에서 바로 발급받는 기능 구현.
- **3단계 (Vertical Expansion):** 법원(등기), 국세청(세무), 병무청 업무 등으로 카테고리 확장 및 기업용 API 출시.

---

## 6. 기대 효과 및 차별점

- **사용자 편의성:** 보안 프로그램 설치 지옥에서 해방, 탐색 시간 80% 단축.
- **접근성 향상:** 정보 취약계층(고령층, 외국인)도 대화만으로 행정 업무 처리 가능.
- **데이터 가치:** 국민들이 어떤 상황에서 어떤 서류를 필요로 하는지에 대한 행정 수요 데이터 확보.

## 7. 개발 환경

### 기술 스택

- **Frontend:** Next.js 14 (App Router), React, TypeScript
- **Styling:** Tailwind CSS, Shadcn UI
- **State Management:** Zustand
- **Data Fetching:** TanStack React Query
- **Validation:** Zod

### 설치 및 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 프로덕션 빌드
npm run build

# 프로덕션 서버 실행
npm start
```
